@3 Динамічно типізований - імена не мають типу.

@3 Високорівневий - не треба заглиблюватися в архітектуру комп'ютера і керувати пам'яттю.

@3 Мультипарадигменний - імперативний, об'єктний, функціональний.

@3 Вільний і відкритий - не має власника і розвивається спільнотою.

@3 З низьким порогом входження - можна швидко почати програмувати

@2 Встановлення 

@3 Відвідати [[https://www.python.org/]] і завантажити дистрибутив.
При встановленні додати шлях до python.exe в системну змінну PATH (просто встановити прапорець).

@3 Запустити інтерпретатор в інтерактивному режимі.
{{
> python
}}

@3 Для отримання допомоги в інтерпретаторі потрібно викликати функцію help() 
і передати їй будь-яке ім'я або в лапках ключове слово .  
{{
help(print)
help(help)
help("return")   
}}

@3 Вихід
{{
quit()
}}

@2 Віртуальне середовище

@3 Віртуальне середовище робить програмний проект незалежним від оточення.
Зазвичай [DIR] то  .venv .
{{
> python -m venv [DIR]
}}

@3 Середовище треба активувати.
Cкрипт активації змінює деякі змінні оточення, і зокрема, змінну PATH.
{{
> [DIR]/Scripts/activate  
}}

@3 Деактивація
{{ 
> [DIR]/Scripts/deactivate  
}}

@3 Перевірка того, який інтерпретатор python викликається.
{{
> Get-Command python
}}
